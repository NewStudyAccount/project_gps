## Context

GPS 仪表盘应用采用深色主题（航空 + 赛车融合风格），使用 Jetpack Compose 构建 UI。当前界面标题使用英文，卫星面板仅显示 PRN 编号，时间显示使用 UTC 时区。

主要涉及文件：
- `DashboardScreen.kt` - 主界面布局
- `SatellitePanel.kt` - 卫星信号面板
- `GpsViewModel.kt` - 数据处理与时区转换

## Goals / Non-Goals

**Goals:**
- 将界面标题和标签汉化，提升中文用户体验
- 重构卫星面板为地球样式，按星座分组展示
- 时间显示改为 UTC+8（北京时间）

**Non-Goals:**
- 不添加真实卫星名称映射（如 NAVSTAR 74）
- 不支持多语言切换
- 不修改卫星信号强度计算逻辑

## Decisions

### 1. 卫星卡片布局

**决策**：采用按星座分组的网格布局，每组上方有虚线轨道连接地球 emoji。

**理由**：
- 分组展示比扁平网格更直观
- 虚线轨道提供视觉层次感
- 地球 emoji 简单易实现，无需自定义 Canvas

**替代方案**：
- 环绕地球的圆形布局 → 实现复杂，小屏幕适配困难
- 扁平网格 + 星座颜色区分 → 视觉层次不够清晰

### 2. 卫星名称格式

**决策**：`{星座缩写} {PRN两位数}`，如 `GPS 01`、`BDS 05`

**理由**：
- 利用现有 `Constellation.label` 字段
- 两位数 PRN 保持卡片宽度一致
- 星座前缀让用户立即识别卫星来源

### 3. 时区处理

**决策**：在 ViewModel 中将 UTC 时间戳转换为 UTC+8

**理由**：
- GPS 原始时间戳为 UTC，转换逻辑放在 ViewModel 层
- 使用 `java.util.TimeZone.getTimeZone("UTC+8")`
- 标签显示 "UTC+8" 而非 "北京时间"（更通用）

### 4. 轨道虚线样式

**决策**：使用 Unicode 虚线字符 `┈` 绘制

**理由**：
- 无需 Canvas 绘制，纯文本实现
- 与现有 MonoFontFamily 风格一致
- 跨设备渲染一致

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| 卫星数量多时卡片拥挤 | 限制每行 6 个，超出换行；卡片尺寸缩小到 56dp |
| 地球 emoji 在不同设备渲染差异 | 仅使用基础 🌍 emoji，避免变体 |
| 虚线轨道在小屏幕上可能断裂 | 使用 `fillMaxWidth()` 确保自适应 |
| 中文标签可能导致布局溢出 | 使用较短的中文词汇，监控布局边界 |
