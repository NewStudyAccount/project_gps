# GPS Dashboard — 专业 GPS 信息展示应用

## 概述

一款 Android 端 GPS 信息展示应用，采用深色仪表盘风格（航空骨架 + 赛车灵魂），实时展示经纬度、海拔、速度、方位、精度等全部定位数据，附带动态罗盘和卫星信号面板。

## 设计风格

**融合方案：航空骨架 + 赛车灵魂**

- **骨架（航空）**：黑底、等宽字体、网格线、信息密度高、面板式布局、HUD 质感半透明叠加层
- **灵魂（赛车）**：琥珀橙主色（#FF8C00）、关键数值微发光（glow）、精度指示渐变色条

## 功能需求

### 1. 核心数据展示（12 个字段，全部展示）

| 字段 | 格式 | 说明 |
|------|------|------|
| 纬度 | `39°54'15.1" N` | 度分秒格式 |
| 经度 | `116°24'26.6" E` | 度分秒格式 |
| 海拔 | `43.5 m` | |
| 速度 | `12.3 km/h` | 支持单位切换（见下文） |
| 方位 | `227° SW` | 真北（磁偏角修正） |
| 精度 | `± 3.2 m` | 带颜色状态条 |
| HDOP | `0.8` | |
| 卫星数 | `12/24` | 已用/可见 |
| 定位类型 | `3D FIX` / `2D` / 无 | |
| 定位源 | `GPS+GLONASS` | |
| UTC 时间 | `14:23:05` | |
| 坐标系 | `WGS-84` | 固定显示 |

### 2. 速度单位切换

- 默认 `km/h`，支持切换为 `m/s` 或 `mph`
- 点击速度卡片弹出选择器，选择后自动收起
- 数值实时换算

### 3. 动态罗盘

- **风格**：基础罗盘，不追求复杂 3D 效果
- **显示**：真北（使用 `GeomagneticField` 修正磁偏角）
- **刻度**：主刻度每 30°，次刻度每 10°，细刻度每 5°
- **方位标签**：N/NE/E/SE/S/SW/W/NW 八方位
- **动画**：阻尼旋转（spring animation），方位变化平滑过渡
- **行为**：刻度盘跟随设备旋转，指针固定指向北方

### 4. 海拔历史图表

- 显示最近 60 秒的海拔变化趋势
- 贝塞尔曲线绘制，底部渐变填充
- 纯展示（内存中环形缓冲区，不做持久化）

### 5. 卫星信号面板

- **默认折叠**：显示总览条（信号强度总览 + 平均 CN0）
- **可展开**：点击展开查看每颗卫星的信号强度
- 展开态按星座分类统计（GPS/GLONASS/BeiDou 等）
- 每颗卫星显示编号和信号强度柱

### 6. 精度颜色指示

| 范围 | 颜色 | 含义 |
|------|------|------|
| ±0~5m | 绿色 #00FF6A | 精度优秀 |
| ±5~15m | 黄色 #FFD700 | 精度一般 |
| ±15m+ | 红色 #FF3D00 | 精度较差 |

## 技术方案

| 项目 | 选择 |
|------|------|
| UI 框架 | Jetpack Compose |
| 自定义绘制 | Canvas（罗盘、图表、信号条） |
| 动画 | Animatable + spring |
| 定位服务 | LocationManager（原生，无 GMS 依赖） |
| 传感器 | SensorManager（加速度计 + 磁力计） |
| 卫星数据 | GnssStatus.Callback（API 24+） |
| 状态管理 | StateFlow |
| 最低 API | 26 (Android 8.0) |

### 数据流

```
LocationManager ──→ LocationFlow ──┐
                                   ├─→ GpsViewModel ──→ GpsDashboardScreen
SensorManager ────→ CompassFlow ──┘     (StateFlow)       (Compose UI)
```

## 非目标（Non-goals）

- 不做轨迹记录 / GPX 导出
- 不做地图叠加
- 不做后台持续定位
- 不做 Android Auto / 平板适配
- 不做多坐标系切换（固定 WGS-84）
