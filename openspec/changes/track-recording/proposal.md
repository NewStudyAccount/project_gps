# 轨迹记录与后台定位 — 功能提案

## 概述

在现有 GPS 仪表盘基础上，新增三大能力：

1. **后台持续定位** — Foreground Service 保持 GPS 在后台运行，通知栏显示实时数据
2. **轨迹录制** — 自适应采样 + 降采样，持久化到 Room 数据库，支持 GPX/KML 导出
3. **轨迹回放与回溯** — Canvas 自绘地图显示轨迹，支持沿原路返回导航指引

三个能力的依赖关系：

```
后台定位 (独立基础)
    │
    ├──▶ 轨迹录制 (依赖后台定位持续运行)
    │        │
    │        └──▶ 轨迹回放 / 回溯 (依赖录制数据)
    │
    └──▶ 仪表盘后台保持 (不录制也想切app后回来继续看数据)
```

## 功能需求

### 1. 后台持续定位

**目标**：app 进入后台后 GPS 采集不中断，通知栏实时显示状态。

**机制**：Foreground Service + 常驻通知

```
┌─────────────────────────────────────────────┐
│  通知栏 (后台运行时)                          │
│                                              │
│  🛰️ GPS 仪表盘 — 后台运行中                   │
│  12▲  3D FIX  速度 12.3 km/h                 │
│  [打开应用]                                   │
│                                              │
├─────────────────────────────────────────────┤
│  通知栏 (录制中)                              │
│                                              │
│  🔴 轨迹录制中                                │
│  00:12:34 · 2.3km · 156点 · 均速 11.2km/h   │
│  [打开应用]              [停止录制]           │
│                                              │
└─────────────────────────────────────────────┘
```

**生命周期**：

```
用户启动 app ──▶ 自动启动 Service ──▶ 通知栏常驻
                                          │
用户切到后台 ──▶ Service 继续运行 ──▶ GPS 持续采集
                                          │
用户回到前台 ──▶ Service 继续运行 ──▶ UI 实时更新
                                          │
用户关闭 app ──▶ stopService() ──▶ 通知消失
```

**权限需求**：

| 权限 | 用途 | 版本要求 |
|------|------|----------|
| `FOREGROUND_SERVICE` | 前台服务基础权限 | Android 9+ |
| `FOREGROUND_SERVICE_LOCATION` | 位置类型前台服务 | Android 14+ |
| `POST_NOTIFICATIONS` | 显示通知 | Android 13+ |
| `ACCESS_FINE_LOCATION` | GPS 定位 | 已有 |
| `ACCESS_BACKGROUND_LOCATION` | 后台定位 | Android 10+ |

### 2. 轨迹录制

**目标**：记录用户移动轨迹，持久化存储，支持暂停/恢复/停止。

**录制状态机**：

```
         startRecording()
    ┌─────────────────────────────────┐
    ▼                                 │
┌───────┐   start()   ┌───────────┐  │
│ IDLE  │────────────▶│ RECORDING │  │
│       │◀────────────│           │  │
└───────┘   stop()    └─────┬─────┘  │
    ▲                       │        │
    │         pause()       │        │
    │          ┌────────────┘        │
    │          ▼                     │
    │    ┌───────────┐               │
    │    │  PAUSED   │───────────────┘
    │    │           │  resume()
    │    └───────────┘
    │          │
    └──────────┘
         stop()
```

**自适应采样策略**：

根据速度自动调整采集频率，平衡精度与电量：

| 状态 | 速度范围 | 采样间隔 | 最小记录距离 | 说明 |
|------|----------|----------|-------------|------|
| 静止 | < 0.5 m/s | 30s | — | 只记录时间戳，不记位移 |
| 步行 | 0.5 ~ 3 m/s | 5s | 5m | 步行/慢跑 |
| 骑行 | 3 ~ 10 m/s | 3s | 10m | 自行车/电动车 |
| 驾驶 | ≥ 10 m/s | 1s | 20m | 汽车/高速移动 |

采样逻辑：**同时满足** 时间间隔 AND 距离阈值 才记录一个新点（静止状态除外，仅按时间记录）。

**降采样**：停止录制时，对轨迹执行 Ramer-Douglas-Peucker 算法压缩：

```
原始轨迹 (1000点)                    压缩后 (120点)
                                     
●─●─●─●─●─●─●─●─●─●                ●─────────●
                                        ╲
    左转 ●─●─●─●─●─●                      ●─────●
              ╲                               ╲
               ●─●─●─●                         ●───●

保留拐弯处关键点，去掉直线段中间点
阈值: 5米 (可配置)
```

### 3. Canvas 自绘地图

**目标**：纯 Compose Canvas 绘制轨迹地图，零外部依赖。

**坐标投影**（小范围线性近似）：

```
GPS (lat, lng) ──▶ Screen (x, y)

screenX = (lng - centerLng) × pixelsPerDegreeLng
screenY = (centerLat - lat) × pixelsPerDegreeLat

其中:
  pixelsPerDegreeLat = viewHeight / latRange
  pixelsPerDegreeLng = viewWidth / lngRange
  centerLat/Lng = 轨迹 bounding box 中心
  latRange/lngRange = bounding box 范围 × 1.2 (留 padding)
```

**视觉风格**（延续暗色仪表盘主题）：

```
┌──────────────────────────────────────────────┐
│                                              │
│   ·  ·  ·  ·  ·  ·  ·  ·  ·  ·  ·  ·  ·  │
│   ·  ·  ·  ·  ·  ·  ·  ·  ·  ·  ·  ·  ·  │
│   ·  ·  · ━━━━━━━━━━━━━━●  ·  ·  ·  ·  ·  │
│   ·  ·  ·  ·  ·  ·  ·  · ╲ ·  ·  ·  ·  ·  │
│   ·  ·  ·  ·  ·  ·  ·  ·  ·╲·  ·  ·  ·  ·  │
│   ·  ·  ·  ·  ·  ·  ·  ·  ·  ●  ·  ·  ·  │
│   ·  ·  ·  ·  ·  ·  ·  ·  ·  ·  ·  ·  ·  │
│                                              │
│   ·  网格点 (#2A2A3A, 低对比度)               │
│   ━  轨迹线 (#FF8C00, 2dp)                   │
│   ●  起点 (#00FF6A) / 当前位置 (#FF8C00 脉冲) │
│                                              │
│   手势: 双指缩放 + 单指拖拽平移                │
│   坐标: 左下角显示经纬度刻度                    │
└──────────────────────────────────────────────┘
```

**交互**：

| 手势 | 操作 |
|------|------|
| 双指捏合 | 缩放地图 |
| 单指拖拽 | 平移视图 |
| 双击 | 重置到轨迹全貌 |
| 单击轨迹点 | 显示该点详情（时间、速度、海拔） |

### 5. 轨迹回溯（沿原路返回）

**目标**：引导用户沿已录制的轨迹原路返回，实时导航指引。

这是最复杂的功能。核心是一套**沿已知路径的导航算法**。

**回溯状态机**：

```
         startBacktrack(track)
    ┌──────────────────────────────┐
    ▼                              │
┌─────────┐   deviated()   ┌───────────┐
│TRACKING │───────────────▶│ DEVIATED  │
│         │◀───────────────│           │
└────┬────┘   returned()   └─────┬─────┘
     │                            │
     │  arrive() / abort()        │  abort()
     ▼                            ▼
┌─────────┐                 ┌─────────┐
│COMPLETE │                 │  IDLE   │
└─────────┘                 └─────────┘
```

**核心算法**：

```
输入:
  T = [P0, P1, ..., Pn]  原始轨迹点序列
  U = 当前用户位置
  heading = 用户当前朝向

Step 1: 找最近轨迹段
  for each segment (Pi, Pi+1):
    d = pointToSegmentDistance(U, Pi, Pi+1)
  nearestSegment = argmin(d)

Step 2: 确定回溯方向
  回溯 = 沿轨迹反向行走
  如果用户在 Pi 附近，下一个目标 = Pi-1

Step 3: 生成转向提示
  计算 (Pi-1 → Pi) 与 (Pi-2 → Pi-1) 的夹角:
  ┌──────────────────────────────────┐
  │ 夹角 < 30°  →  "继续直行"        │
  │ 30°~120° 左 →  "前方左转"        │
  │ 30°~120° 右 →  "前方右转"        │
  │ ≥ 120°     →  "大转弯 / 掉头"    │
  └──────────────────────────────────┘

Step 4: 偏离检测
  d = 点到最近轨迹段的距离
  ┌──────────────────────────────────┐
  │ d < 30m  →  正常 (绿色指示)      │
  │ 30~50m   →  接近偏离 (黄色警告)  │
  │ d > 50m  →  偏离 (红色告警)      │
  └──────────────────────────────────┘

Step 5: 到达终点
  distance(U, 轨迹起点) < 20m → 回溯完成
```

## 技术方案

| 项目 | 选择 | 说明 |
|------|------|------|
| 后台定位 | Foreground Service | Android 标准方案 |
| 数据库 | Room (SQLite) | 与现有 Flow 架构无缝衔接 |
| 地图绘制 | Compose Canvas | 零外部依赖，延续自绘风格 |
| 导出格式 | GPX + KML | 纯 XML 生成，无第三方库 |
| 采样策略 | 自适应 (时间+距离) | 省电 + 保留关键细节 |
| 降采样 | Ramer-Douglas-Peucker | 经典轨迹压缩算法 |
| 回溯算法 | 最近段投影 + 方向判断 | 点到线段距离 + 夹角计算 |

## 非目标（Non-goals）

- **不做地图瓦片/底图** — 纯 Canvas 自绘轨迹，不显示街道
- **不做社交分享** — 只做文件导出，不做在线分享
- **不做多轨迹对比** — 单条轨迹回放/回溯
- **不做轨迹编辑** — 不支持手动修改轨迹点
- **不做实时位置共享** — 不涉及网络通信
- **不做运动统计详情** — 不做心率、步频等运动数据（仅 GPS 数据）

## 风险与注意事项

| 风险 | 说明 | 缓解措施 |
|------|------|----------|
| 厂商后台限制 | 小米/华为/OPPO 会激进杀后台 | 引导用户关闭电池优化 + 加入白名单 |
| Android 14+ 前台服务限制 | 必须声明 foregroundServiceType | manifest 中配置 `location` 类型 |
| GPS 隧道/室内信号丢失 | 轨迹出现跳跃或中断 | 记录 accuracy，丢弃精度差的点 |
| 长时间录制存储增长 | 1小时约 360-720 点 | 自适应降频 + RDP 压缩 + 提供清理功能 |
| Canvas 地图性能 | 大量轨迹点绘制卡顿 | 视口裁剪（只绘制可见区域）+ 降采样 |
| 回溯偏离后重新规划 | 用户大幅偏离后如何重新引导 | 先指引到最近轨迹点，再继续回溯 |
